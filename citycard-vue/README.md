# 城市卡牌游戏 — Vue 3 重构版

从原版 45000+ 行单文件 HTML (`citycard_web.html`) 重构为模块化 Vue 3 应用。

**状态**: 核心可玩（~95%）
**最后更新**: 2026-03-02

---

## 快速开始

```bash
cd citycard-vue
npm install
npm run dev          # 默认 http://localhost:5175/
```

进入后 **主菜单 → 开始游戏 → 选择模式（单机/在线） → 开始对战**。

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器 |
| `npm run build` | 生产构建 |
| `npm run preview` | 预览生产构建 |
| `npm run test` | 运行测试（65/65 通过） |
| `npm run test:ui` | 测试（带 UI 面板） |
| `npm run test:coverage` | 测试覆盖率报告 |

清除缓存（遇到问题时）：

```bash
rm -rf node_modules package-lock.json .vite && npm cache clean --force && npm install
```

## 架构设计

```
展示层  Vue Components（.vue 文件）
  ↓
业务层  Composables（useGameLogic, skills, skillCore, citySkills）
  ↓
数据层  Pinia Store（gameStore.js — 响应式全局状态）
  ↓
工具层  Constants / Utils / Data
```

### 三层技能架构

```
核心效果层 (skillCore/)     纯游戏逻辑，模式无关，参数化传入
  ↓
模式适配层 (adapters/)      主持人模式 / 玩家模式差异处理
  ↓
统一接口层 (useSkillEffects) 自动分发到对应适配器
```

**设计原则**:
- 核心层是纯函数，无 UI/Firebase 依赖，只接收参数返回 `{ success, message, data }`
- 玩家模式和主持人模式共享核心逻辑，仅适配层不同
- 技能分类：A类（单方立即生效）、B类（双方确认多阶段）、C类（需额外系统支持）

### 城市标识迁移（已完成）

已从数组索引迁移到**城市名称（name-based）**字典结构（33个核心文件）：

```javascript
// 之前
player.cities = [{ name:'北京市', hp:10000 }, ...]
player.centerIndex = 0

// 之后
player.cities = { '北京市': { name:'北京市', hp:10000 }, ... }
player.centerCityName = '北京市'
```

所有状态键统一使用 `[playerName][cityName]` 格式（protections, anchored, ironCities 等）。

## 游戏功能

### 游戏模式

| 模式 | 玩家数 | 说明 |
|------|--------|------|
| **2P** | 2 | 双人对战 |
| **3P** | 3 | 三人混战 |
| **2v2** | 4 | 团队对战 |

每种模式均支持**单机（vs AI）**和**在线对战**。

### 游戏流程

**单机模式**: 输入昵称 → 选择模式(2P/3P/2v2) → 抽取城市 → 选择中心城市 → 回合制对战

**在线模式**: 创建/加入房间(4-6位数字) → 等待玩家就绪 → 选择中心城市 → 每回合部署城市+使用技能 → 战斗计算同步

**观战模式**: 房间满员后加入的玩家自动成为观战者，可查看日志但不参与游戏。

### 游戏规则

- 每位玩家抽取 10 座城市（2v2 为 7 座），选 1 座中心城市
- 每回合获得 3 金币，使用金币购买技能
- 城市战力 = 当前 HP + 红色技能加成（每级 x1.5 倍）
- 伤害按 HP 升序分配给防守方城市
- 中心城市被摧毁即淘汰（特殊技能例外）
- 城市连续出战触发疲劳（HP 减半），越战越勇/既来则安可豁免
- 同省规则（仅2P）：同省城市同时出战 → 撤退；一方有省会 → 非省会归顺

## 主持人模式使用

主持人模式用于主持线下多人游戏：

1. **初始化**: 选择玩家数量(2/3/4) → 设置城市数(3-10，推荐6) → 初始化
2. **玩家面板**: 查看城市详情、金币、存活数；手动 +3 金币
3. **技能面板**: 选择玩家 → 选战斗/非战斗技能 → 查看可用状态 → 使用
4. **战斗面板**: 为每个玩家选择出战城市 → 确认部署 → 执行战斗 → 查看结果
5. **城市编辑器**: 实时修改 HP（滑块/输入）、技能等级(红/绿/蓝/黄 0-10)、保护状态、复活/击杀/重置
6. **回合管理**: 下一回合（自动+3金币）、保存/加载游戏

### 战力计算

```
城市战力 = 当前HP + 红色技能加成
红色等级加成: 1级=+1000, 2级=+1500, 3级=+2250, ... (每级x1.5)
净伤害 = 攻击力 - 防御力 (→ 护盾处理 → 按HP升序分配)
```

## 常见问题

**Q: 启动报错？** — 尝试 `rm -rf node_modules && npm install`，或清除 `.vite` 缓存。

**Q: 技能按钮灰色？** — 检查金币是否足够、是否在冷却中、使用次数是否用尽。

**Q: 在线连接失败？** — 检查 Firebase 配置是否正确，确保网络连接正常。

**Q: 游戏卡在"战斗计算中"？** — 检查控制台错误，尝试刷新页面。

**Q: 存档在哪？** — 浏览器 LocalStorage，清除浏览器数据会丢失。用"导出"功能备份为 JSON。
